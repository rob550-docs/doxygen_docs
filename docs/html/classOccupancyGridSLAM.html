Detailed Description

OccupancyGridSLAM runs on a thread and handles mapping.

LCM messages are assumed to be arriving asynchronously from the runSLAM thread. Synchronization between the two threads is handled internally.

void OccupancyGridSLAM::runSLAM ( void  )

runSLAM enters an infinite loop where SLAM will keep running as long as data is arriving. It will sit and block forever if no data is incoming.

This method should be launched on its own thread.
